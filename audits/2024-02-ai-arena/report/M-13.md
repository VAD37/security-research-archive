# `FighterFarm.reRoll()` does not check if token have correct `fighterType`. Bypassing maxReroll limit

## Impact

User of wrong `fighterType` can reroll beyond limit of `maxRerollsAllowed[fighterType]`.

## Proof of Concept

[There is only 2 fighters type ID.](https://github.com/code-423n4/2024-02-ai-arena/blob/f2952187a8afc44ee6adc28769657717b498b7d4/src/FighterFarm.sol#L127)
`/// @param fighterType Type of fighter either 0 or 1 (champion or dendroid).`

[Each type can reroll max 3 times.](https://github.com/code-423n4/2024-02-ai-arena/blob/f2952187a8afc44ee6adc28769657717b498b7d4/src/FighterFarm.sol#L35-L36)

```
    /// @notice The maximum amount of rerolls for each fighter.
    uint8[2] public maxRerollsAllowed = [3, 3];
```

After admin increase generation, reroll limit increase by 1.
<https://github.com/code-423n4/2024-02-ai-arena/blob/f2952187a8afc44ee6adc28769657717b498b7d4/src/FighterFarm.sol#L125-L134>

So if admin increase generation of champion to 10, while dendroid is 5. Champion can reroll 10 times.
While dendroid can reroll 5 times.

Because `reRoll()` function does not check if token have correct fighterType as user input.
User can reroll dendroid 10 times despite being wrong type.
<https://github.com/code-423n4/2024-02-ai-arena/blob/f2952187a8afc44ee6adc28769657717b498b7d4/src/FighterFarm.sol#L370-L391>

## Tools Used

<https://github.com/code-423n4/2024-02-ai-arena/blob/f2952187a8afc44ee6adc28769657717b498b7d4/src/FighterFarm.sol#L372>

## Recommended Mitigation Steps

Change fighterType to enum.

```solidity
    /// @notice Rolls a new fighter with random traits.
    /// @param tokenId ID of the fighter being re-rolled.
    /// @param fighterType The fighter type.
    function reRoll(uint8 tokenId, uint8 fighterType) public {
        require(msg.sender == ownerOf(tokenId));//@audit can reroll despite wrong fighter type
        require( (fighters[tokenId].dendroidBool?1:0) == (fighterType), "Fighter type does not match");
    }   
```
