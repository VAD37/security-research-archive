# Missing Utilization Rate Cap

## Description

The `utilizationRate()` function calculates `borrows / (availableLiquidity + borrows)` and, by design, will never exceed 100%.
However, there is no explicit clamp, so if future changes ever allow `borrows > totalLiquidity`, the returned rate could exceed 100% and lead to unintended behavior in rate calculations cause borrow rate rise above maximum rate.

## Recommendation

After computing the raw rate, enforce a maximum of 100% (i.e. Precision.FACTOR1E18) as a defense-in-depth measure:

```solidity
function utilizationRate(DataTypes.ReserveData storage reserve) internal view returns (uint256 rate) {
    (uint256 total, uint256 borrows) = totalLiquidityAndBorrows(reserve);
    if (total > 0) {
        rate = borrows.mul(Precision.FACTOR1E18).div(total);
+       if (rate > Precision.FACTOR1E18) {
+           rate = Precision.FACTOR1E18;
+       }
    }
    return rate;
}
```
