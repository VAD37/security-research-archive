# `AccountingManager` No minimum deposit/withdrawal allowing users spamming small deposit/withdrawal to waste vault manager gas

Vault Manager need to handle all deposit/withdrawal queue array one by one, if user spamming small deposit, it will waste manager gas and time to handle the queue.

Also withdrawal function accept withdraw zero share.

## Impact

Basic gas DOS by spamming small deposit.

## Proof of Concept

When user withdraw their share. There is no zero check or minimum share check. It just add user withdrawal to queue array.
<https://github.com/code-423n4/2024-04-noya/blob/9c79b332eff82011dcfa1e8fd51bad805159d758/contracts/accountingManager/AccountingManager.sol#L303-L315>

Then Vault Manager or vault keepers must go through 4 different function that is very gas expensive to handle each user personal queue.
CalculateShare -> startWithdrawal -> fulfillWithdrawal -> execute withdrawal for each user.
<https://github.com/code-423n4/2024-04-noya/blob/9c79b332eff82011dcfa1e8fd51bad805159d758/contracts/accountingManager/AccountingManager.sol#L316-L445>

## Recommended Mitigation Steps

Minimum deposit and withdrawal queue should map to user address.
Allowing replacing user deposit in queue instead of adding new one in queue.
