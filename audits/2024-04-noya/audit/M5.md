# `SwapAndBridgeHandler` post execution slippage check was not implemented. Slippage check completely depend on Li.Fi exchange external call

Swap bridge handle SwapRequest and figure out minimum output token amount and pass that to sub-route contract which is `LifiImplementation.sol`.
This swap implementation failed to verify swap data slippage too.

## Impact

There is slippage input but no Slippage check.
No verification for swapping call from Maintainer.

## Proof of Concept

There is minimum slippage when swapping.
<https://github.com/code-423n4/2024-04-noya/blob/9c79b332eff82011dcfa1e8fd51bad805159d758/contracts/helpers/SwapHandler/GenericSwapAndBridgeHandler.sol#L98-L113>

SwapHandler will call into sub route which is LifiImplementation to swap token.
https://github.com/code-423n4/2024-04-noya/blob/9c79b332eff82011dcfa1e8fd51bad805159d758/contracts/helpers/SwapHandler/GenericSwapAndBridgeHandler.sol#L115

But this implementation Li.Fi contract failed to check minimum swap amount inside external call swap function too.
<https://github.com/code-423n4/2024-04-noya/blob/9c79b332eff82011dcfa1e8fd51bad805159d758/contracts/helpers/SwapHandler/Implementaions/LifiImplementation.sol#L118-L122>

This result in no slippage check inside Noya Codebase.

All slippage will be handle by Li.Fi exchange itself.

        _amountOut = ISwapAndBridgeImplementation(swapImplInfo.route).performSwapAction(msg.sender, _swapRequest);
