# Emergency role in Registry cannot call `onlyVaultMaintainer` modifier

Registry `EMERGENCY_ROLE` cannot use `onlyVaultMaintainer` modifier due to error in boolean logic.

## Impact

Emergency role cannot use some Registry function `addConnector`, `updateConnectorTrustedTokens`,`removeTrustedPosition`.
Emergency address suppose to be allowed to do all the operation that vault maintainer can do.

## Proof of Concept

Emergency role have same power as vault governor.
Governor have permission to change all vault address.
It is expected that `EMERGENCY_ROLE` must have same power as vault maintainer role too.

`onlyVaultMaintainer` have || operation check which mean if msg.sender has role emergency, it will revert.
This is different from other modifier check as seen below

```solidity
    // -------------------- MODIFIERS --------------------
    //@ only maintainer
    modifier onlyVaultMaintainer(uint256 _vaultId) {
        if (msg.sender != vaults[_vaultId].maintainer || hasRole(EMERGENCY_ROLE, msg.sender) == false) {
            revert UnauthorizedAccess();
        }//@audit || operation here is incorrect. It should be &&
        _;
    }
    //@emergency role can call this. maintainer role can call this
    modifier onlyVaultMaintainerWithoutTimeLock(uint256 _vaultId) {
        if (msg.sender != vaults[_vaultId].maintainerWithoutTimeLock && hasRole(EMERGENCY_ROLE, msg.sender) == false) {
            revert UnauthorizedAccess();
        }
        _;
    }
    //@ vault governer will pass. emergency role will pass
    modifier onlyVaultGoverner(uint256 _vaultId) {
        if (msg.sender != vaults[_vaultId].governer && hasRole(EMERGENCY_ROLE, msg.sender) == false) {
            revert UnauthorizedAccess();
        }
        _;
    }
```
