# `NoyaValueOracle.getValue()` input wrong token address in loop. Return wrong oracle value if quote price more than 2 oracles

`NoyaValueOracle.getValue()` request multiple oracles in a loop to convert price from assetToken to baseToken.
For example UNI->WETH->USDC->$USD.

Currently, the loop mistakenly only quoting first token in array instead of moving to next token in path.

This result in UNI->WETH, UNI->USDC, UNI->$USD.
Instead of UNI->WETH, WETH->USDC, USDC->$USD.

## Impact

`NoyaValueOracle` return wrong value if quote token have more than 2 oracle path.
Error in price,decimal conversion between oracles due to wrong input.

## Proof of Concept

```solidity

    function _getValue(address asset, address baseToken, uint256 amount, address[] memory sources)
        internal
        view
        returns (uint256 value)
    {
        uint256 initialValue = amount;
        address quotingToken = asset;
        for (uint256 i = 0; i < sources.length; i++) {//@this loop allow getOracle value for BTC/WETH/USDC/BaseToken
            initialValue = _getValue(asset, sources[i], initialValue);//@ asset here should be quotingToken
            quotingToken = sources[i];
        }
        return _getValue(quotingToken, baseToken, initialValue);
    }
```

When one of oracle quote wrong token address, then `initialValue` will also be wrong.
After loop quoting next oracle, because input amount `initialValue` is wrong, then return value will also be wrong.

## Tools Used

<https://github.com/code-423n4/2024-04-noya/blob/9c79b332eff82011dcfa1e8fd51bad805159d758/contracts/helpers/valueOracle/NoyaValueOracle.sol#L81-L93>

## Recommended Mitigation Steps

`_getValue(asset, sources[i], initialValue);` should be `_getValue(quotingToken, sources[i], initialValue);`
