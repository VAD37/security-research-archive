# Uniswap TransferAllowance Permit2 signature hjack

<https://docs.uniswap.org/contracts/permit2/reference/allowance-transfer#security-considerations>
https://github.com/dragonfly-xyz/useful-solidity-patterns/tree/main/patterns/permit2#integrating-permit2

Uniswap Docs state pretty clear that all contracts use `SignatureTransfer` and `PermitTransferFrom` should always use `msg.sender` as signature owner.
Otherwise, exploiter can read signature from mempool and use the signature allowance as their own.

This was not implemented in `PositionAction.sol`, `SwapAction.sol`.
