## Summary

Removing and re-adding a rewards token in `DefiAppStaker.sol` resets the global reward index without updating user indices, allowing previous stakers to drain all remaining rewards.

## Finding Description

The staking contract uses a flywheel model to track rewards by caching the token distribution speed per second and maintaining a global rewards index. When users claim rewards, their last claim is recorded based on this global index. However, if an admin removes a rewards token, the global index resets while individual user indices remain unchanged. Consequently, when the same token is re-added, the discrepancy between the reset global index and the stored user indices allows previous stakers to claim additional rewards, effectively draining all tokens held in `DefiAppStaker.sol`.

## Impact Explanation

High: Under specific condition, anyone can drain all remaining rewards tokens from the contract.

## Likelihood Explanation

Low: This vulnerability requires an admin to intentionally remove and then re-add the same token as a reward, a scenario that is unlikely under normal circumstances.
